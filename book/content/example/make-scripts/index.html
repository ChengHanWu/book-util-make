make scripts

如何使用 make 來達到像是 「npm scripts」或是「composer scripts」的功能

基本概念

撰寫「Makefile」

「Makefile」內容如下

.PHONY: usage status deploy

usage:
    echo make command

status:
    echo show status

deploy:
    echo deploying...

執行指令

執行

$ make

顯示

echo make command
make command

執行

$ make usage

顯示

echo make command
make command

這個結果跟上面直接下「make」是一樣的，因為若沒給第一個參數，就會找第一個Rule，「usage」在這個範例是放在第一個，所以「make」和「make usage」結果一樣。

執行

$ make status

顯示

echo show status
show status

執行

$ make deploy

顯示

echo deploying...
deploying...

執行

$ make deploy -s

顯示

deploying...

這裡加上「-s」，就不會把指令也印出來，請參考「GNU make / 5.2 Recipe Echoing」。

或是改寫「Makefile」

.PHONY: usage status deploy

usage:
    @echo make command

status:
    @echo show status

deploy:
    @echo deploying...

關於「.PHONY」的用法，請參考「GNU make / 4.6 Phony Targets」

「完整範例」

把指令分散到「Shell Script」

上面的寫法，只需要一個「Makefile」搞定。

而我個人慣用把這些指令拆解到個別的「Shell Script」去，就會如同下面的結構。

./
├── bin
│   ├── deploy.sh
│   ├── status.sh
│   └── usage.sh
└── Makefile

1 directory, 4 files

撰寫「Makefile」，內容如下

THE_MAKEFILE_FILE_PATH := $(abspath $(lastword $(MAKEFILE_LIST)))
THE_BASE_DIR_PATH := $(abspath $(dir $(THE_MAKEFILE_FILE_PATH)))
THE_BIN_DIR_PATH := $(THE_BASE_DIR_PATH)/bin

.PHONY: usage status deploy

usage:
    @$(THE_BIN_DIR_PATH)/usage.sh

status:
    @$(THE_BIN_DIR_PATH)/status.sh

deploy:
    @$(THE_BIN_DIR_PATH)/deploy.sh

撰寫「bin/usage.sh」，內容如下

#!/usr/bin/env bash

usage()
{
    echo ""
    echo "Usage: make [command]"
    echo
    cat <<EOF
Ex:
$ make
$ make usage
$ make status
$ make deploy
EOF
}

usage

撰寫「bin/status.sh」，內容如下

#!/usr/bin/env bash

echo show status

撰寫「bin/deploy.sh」，內容如下

#!/usr/bin/env bash

echo deploying...

執行指令

執行

$ make

或是執行

$ make usage

顯示

Usage: make [command]

Ex:
$ make
$ make usage
$ make status
$ make deploy

執行

$ make status

顯示

show status

執行

$ make deploy

顯示

deploying...

「完整範例」

傳參數

尚未研究傳參數的方式，因為在工作流程上，就是要把一些常用的動作包在一起，所以就直接把參數寫死在script上了。

只需要下「make [command]」就行了。
